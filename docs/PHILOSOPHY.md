# PromptWash Philosophy

PromptWash follows a set of guiding principles.

---

## Local-First

Prompt processing should happen locally whenever possible.

---

## Minimal Dependencies

The tool should avoid unnecessary libraries or cloud dependencies.

---

## Stable CLI Interface

Users should only need to remember a small set of commands.

---

## Prompt Hygiene

Prompts should be:

- clear
- structured
- reproducible

---

## Extensibility

PromptWash should grow through modules rather than breaking existing workflows.
