# PromptWash UI Metrics

PromptWash optimizes for three primary user outcomes.

## 1. Time to First Structure

How quickly a user sees meaningful structure extracted from messy input.

Example signals:

- Goal detected
- Audience detected
- Constraints detected

Target: near-instant feedback.

## 2. Prompt Improvement Rate

How often the user ends up with a better prompt.

Signals:

- Refactors accepted
- Structure added
- Lint warnings reduced
- Evaluation score improved

Goal: users consistently leave with improved prompts.

## 3. Depth Activation Rate

How often users intentionally explore deeper tools.

Examples:

- Opening Variant Drawer
- Running prompts
- Opening Prompt Debugger
- Running experiments

Goal: the workspace remains calm while deeper tools stay discoverable.

## Success Definition

PromptWash succeeds when users:

- Quickly see structure
- Leave with a better prompt
- Go deeper only when they want to
